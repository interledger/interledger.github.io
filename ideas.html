Google Summer of Code

Project Ideas

Here you can find ideas for a project that can be done as part of the Google Summer of Code 2017 program.

Integrate Interledger into an App or Protocol

Imagine the Web without ads, Torrents where seeders compete to upload files to you as fast as possible, or the Internet without DDoS attacks. Interledger (and especially Interledger-based micropayments) can be used to enable each of these visions.

Examples of this are:

  • Pay for content as you browse the web instead of seeing ads.
  • Pay for downlaods from torrent seeders (and pay the original content author).
  • Pay to get guaranteed delivery of an email (skip SPAM filters).
  • Pay for public wifi as you use it.

Requirements

Integrate interledger payments into an existing technology to demonstrate how micro-payments could change the market dynamics and incentives around an exisiting ecosystem.

Any

Evan Schwartz

Build an Interledger mobile wallet.

Numerous mobile wallets exist today but none use an open protocol to make payments. It should be possible to easily send money to someone using Interledger.

Requirements

Build a mobile app that allows a user to send money to somebody else (who may not be connected to Interledger).

Social Media Integration

When users run an ILP-Kit node the first thing they need to do is find other people they trust that are also running nodes so they can peer with them.

If they could link a social media profile and let ILP-Kit access their list of connections then we could co-ordinate a "match-making service" of ILP-Kit administrators through their existing social-media connections.

Requirements

Build a service that:

  • stores the social media graph of ILP-kit users
  • looks for connections on the graph to other ILP-kit users
  • suggests other ILP-kits that a user may want to peer with

Any, although Javascript preferred.

ILP-enable a Ledger

Interledger has specific service requirements from any payment network/ledger that becomes part of the network. The most common way to ILP-enable a ledger is to write a plugin/adaptor for that ledger that conforms to the Interledger specifications

The community has been particulalry excited about integrating the Bitcoin Lightning and Ethereum Raden netwokrs into the Interledger so experience with either of these would be a big bonus.

Requirements

Build a new Javascript ledger plugin that confirms to the ledger plugin specification or build a Java ledger adaptor based on the Java interface.

Tech

Javascript or Java

Mentors

For Java adaptors: Adrian Hope-Bailie

For Javascript plugins: Evan Schwartz

ILP-enable Chain Core

Chain Core is a multi-asset cryptographic ledger (a.k.a. blockchain) inspired by Bitcoin's UTXO model: https://chain.com/technology/

The goal of this project is to ILP-enable Chain Core by writing a plugin ("ilp-plugin-chain") using Chain's Node.js SDK.

Requirements

Build a Javascript ledger plugin that confirms to the ledger plugin specification using Chain Core's Node.js SDK

Javascript

Easy deployment

Many potential ILP-kit users are not technical but have available liquidity that they can add to the network. We need to make it really easy for them to deploy and configure a node.

Possible solutions are "packaged applications" that are available on many cloud hosting platforms. Ideally the user experience for deploying ILP-kit would be to run through the deployment flow defined on the hosting platform following which a fully-functional ILP-kit is up and running at a URL of the deployer's choosing.

Requirements

Package ILP-kit such that it can be easily deployed and configured by non-technical users.

Tech

Scripting and automation. Possibly Javascript if changes to ILP-kit are required.

Monitoring

ILP-Kit currently outputs a number of logs which are useful for technical debugging but it would also be valuable to produce instrumentation data that can be used to monitor things like liquidity, performance, reach etc.

It would also be useful to have a mechanism to aggregate (and anonymize) this data and submit it to a central monitoring service that can monitor the health of the entire network.

Requirements

Any or all of the following:

  • Add instrumentation output to ILP modules that would make collection of operational data easier.
  • Build a monitoring and rules engine (or adaptor for an existing one) that can be used for operating an ILP node
  • Build an data collection and aggreagtion service for ILP data that can be used to monitor the health of the network.

Tech

Javascript for updates to ILP-kit. Any for other deliverables.
